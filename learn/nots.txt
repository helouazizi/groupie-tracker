### Common Misconceptions
## Assuming Safety:
Many programmers assume that simply writing code in C makes it safe 
from exploitation. However, C provides low-level access to memory and
system resources, which can lead to vulnerabilities if not properly managed.

## Buffer Overflows:
 One of the most common vulnerabilities in C 
 programs is the buffer overflow. This occurs when a 
 program writes more data to a buffer than it can hold, 
 potentially overwriting adjacent memory. Hackers exploit 
 this to execute arbitrary code or crash the program.

 ##Pointer Misuse: 
 C allows direct manipulation of memory through pointers, 
 which can lead to security issues if pointers are not used 
 carefully. Dereferencing null or invalid pointers can lead to 
 crashes, while manipulating memory can lead to unauthorized access 
 to sensitive data.
